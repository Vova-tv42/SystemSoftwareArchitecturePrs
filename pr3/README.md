# Практична робота №3: Дослідження обмежень ресурсів у середовищі Docker

## Склад проєкту

- `task1.sh` - Bash-скрипт для дослідження лімітів відкритих файлів.
- `task2.sh` - Bash-скрипт для роботи з утилітою `perf`.
- `task3.c` - програма-симулятор кидання кубика з обмеженням на розмір файлу.
- `task4.c` - генератор лотерейних комбінацій з обмеженням часу ЦП.
- `task5.c` - програма для безпечного копіювання файлів з перевіркою лімітів.
- `task6.c` - демонстрація переповнення стеку через рекурсію.
- `task_v9.c` - дослідження обмеження динамічної пам'яті (Варіант 9).
- `build.sh` - скрипт для автоматизації компіляції.

---

## Документація програм

### task1.sh

**Увага:** Очікується, що користувач має встановлений **Docker**.

Програма виконує послідовність команд `ulimit` для дослідження м'яких (Soft) та жорстких (Hard) лімітів кількості відкритих файлів. Вона демонструє, як система дозволяє змінювати ліміти в певних межах та як блокуються спроби виходу за жорстку межу.

**Можливі помилки:**

### task2.sh

**Увага:** Очікується, що користувач має встановлений **Docker** та утиліту **perf**.

Скрипт запускає аналіз продуктивності програми `task4` за допомогою утиліти `perf stat`. Це дозволяє отримати детальну статистику про використання інструкцій та циклів процесора до моменту зупинки програми через ліміт часу.

### task5.c

Програма для копіювання вмісту одного файлу в інший. Вона демонструє безпечне копіювання з урахуванням обмеження `RLIMIT_FSIZE` (1024 байти) для вихідного файлу.

**Приклад запуску:**

```bash
./task5 <source_file> <destination_file>
```

**Аргументи запуску:**

- `source_file` - назва або шлях до джерельного файлу, який потрібно прочитати.
- `destination_file` - назва або шлях до файлу призначення, куди будуть записані дані.

**Можливі помилки:**

- **Програма приймає 2 аргументи:** Помилка, якщо кількість переданих аргументів не дорівнює двом.
- **Не вдалося відкрити файл ... для читання/запису:** Помилка при відсутності файлу або недостатніх правах доступу.
- **SIGXFSZ:** Помилка виникає, якщо вхідний файл більший за 1 КБ - програма повідомляє про досягнення ліміту під час копіювання.

### task_v9.c

Програма для дослідження обмеження динамічної пам'яті. Вона обмежує сегмент даних процесу (`RLIMIT_DATA`) до 10 МБ та циклічно виділяє пам'ять через `malloc`.

**Результат виконання:**

<img width="605" height="353" alt="image" src="https://github.com/user-attachments/assets/ec5bbae7-dae0-4284-b6c9-eba43411862f" />


**Можливі помилки:**

- **malloc() повернув NULL:** При вичерпанні ліміту пам'яті `malloc` повертає порожній вказівник. Програма обробляє цей випадок та виводить: `Помилка: malloc() повернув NULL. Досягнуто межі RLIMIT_DATA.`.
