# Практична робота №6: Інструменти налагодження для проблем з пам'яттю

## Налаштування та запуск

Для компіляції програми використовується скрипт `build.sh`. Він створює дві версії виконуваного файлу: з підтримкою PIE та без неї.

**Команда для компіляції:**
```bash
./build.sh
```

**Команди для запуску та тестування ASLR:**
```bash
# Запуск з увімкненим ASLR (стандартна поведінка)
./task_v9_pie

# Запуск з вимкненим ASLR (через setarch)
setarch $(uname -m) -R ./task_v9_pie

# Порівняння з версією без PIE
./task_v9_no_pie
```

---

## Використання програми

Програма `task_v9.c` виводить адреси різних сегментів пам'яті:
1. **Текстовий сегмент (main)** - де зберігається код.
2. **Сегмент даних (global)** - де зберігаються глобальні змінні.
3. **Стек (local)** - адреса локальної змінної.
4. **Купа (heap)** - адреса динамічно виділеної пам'яті.
5. **Бібліотечна функція (printf)** - адреса функції з libc.

---

### Роль PIE:
PIE (Position Independent Executable) дозволяє завантажувати основний код програми у довільне місце в пам'яті. Без цього параметра ASLR не може рандомізувати адреси текстового сегменту та сегменту даних самої програми.
