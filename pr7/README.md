# Практична робота №7: Дослідження, моделювання та нестандартні підходи до аналізу процесів, файлових систем, безпеки та ресурсів в Linux

## Налаштування та запуск

**Компіляція всіх завдань:**

Для компіляції програми використовується скрипт build.sh.
```
./build.sh
```

## Використання файлів програм та пояснення

### task1.c
Програма використовує `popen()`, щоб передати вивід команди `who` до команди `more`. Вона відкриває один потік для читання виводу `rwho` та інший для запису в `more`, копіюючи дані між ними.

**Примітка:** Замість `rwho` було використано сучасну утиліту `who`.

### task3.c
Програма друкує всі рядки з файлу, що містять слово, передане як перший аргумент.

- **Приклад запуску:** `./task3 <word> <file_name>`

**Аргументи запуску:**
- `word` — слово, яке необхідно знайти (обов'язковий).
- `file_name` — назва або шлях до файлу для пошуку (обов'язковий).

### task4.c
Виводить список файлів із паузою кожні 20 рядків. Очікує натискання клавіші [Enter] для продовження.
- **Приклад запуску:** `./task4 <file1> [file2 ...]`

**Аргументи запуску:**
- `file1` — шлях до першого файлу для перегляду (обов'язковий).
- `file2 ...` — шляхи до додаткових файлів (опціонально).

### task10.c
Генерує випадкові числа з плаваючою точкою у діапазонах [0.0, 1.0] та [0.0, n].

**Аргументи запуску:**
- `n` — верхня межа діапазону від 0.0 до n (опціонально, за замовчуванням 100.0).

### task_v9.c
